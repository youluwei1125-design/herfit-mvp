## 1. 产品概述

### 1.1 产品名称

HerFit｜AI 驱动的女性周期友好训练助手

### 1.2 一句话定位

HerFit 是一款面向女性健身用户的 AI 日常训练决策助手，基于用户的生理周期阶段、身体状态和训练场景，生成更适合当天执行的个性化训练建议。

### 1.3 当前版本定位

当前 V0.1 MVP 重点验证的是：

> 用户是否需要一个能够结合「周期阶段 × 身体状态 × 训练场景」来辅助判断“今天适合练什么”的 AI 训练助手。

当前版本不是完整商业化 App，也不是医疗产品，而是一个用于验证核心产品逻辑、用户路径和 AI 推荐框架的可交互 MVP。

### 1.4 产品愿景

HerFit 希望从“每日训练推荐工具”逐步发展为“周期友好的自适应训练计划助手”。未来产品将通过周期记录、训练记录、用户反馈和个性化偏好，持续优化用户的训练计划，使用户在不同身体状态下都能获得更安全、更可执行、更可持续的训练建议。

## 2. 问题背景

女性用户在日常健身中经常面临一个具体但高频的问题：

> 今天我到底适不适合训练？如果适合，应该练什么、练多久、练多强？

现实中，女性用户的训练状态并不是稳定不变的。生理周期、痛经、腹胀、疲劳、睡眠质量、情绪状态、可用时间和训练环境都会影响当天是否适合训练，以及适合什么强度的训练。

现有产品通常存在两类断层：

1. 训练类产品更关注动作库、计划执行和训练记录，但较少考虑生理周期和身体状态对训练安排的影响。
2. 周期 / 健康类产品更关注经期记录、症状追踪和健康内容，但通常不会进一步将这些状态转化为具体训练决策。

因此，HerFit 的机会点不是再做一个普通训练记录工具，也不是再做一个经期记录工具，而是连接两者之间的空白：

> 将周期信息和身体状态转化为当天可执行的训练建议。

## 3. 目标用户

### 3.1 核心用户

HerFit 面向有健身需求、但希望训练安排更贴合身体状态的女性用户，尤其是经常通过 B 站、小红书、YouTube 等平台进行居家跟练的用户。

### 3.2 用户特征

- 年龄：18-35 岁女性为主
- 训练场景：居家、宿舍、健身房、办公室、出差酒店等
- 训练方式：自重训练、轻器械训练、跟练视频为主
- 训练经验：新手到有一定基础的用户
- 核心诉求：想规律训练，但经常受身体状态、周期变化和时间安排影响
- 内容习惯：习惯使用免费跟练视频，但经常需要花时间筛选

### 3.3 核心痛点

| 痛点 | 用户表现 | HerFit 解决方向 |
|---|---|---|
| 不知道今天该不该练 | 周期变化、疲劳或不适时犹豫是否训练 | 基于周期和身体状态给出当天训练建议 |
| 不知道练什么 | 视频太多，需要自行搜索和筛选 | 根据状态生成训练类型，并提供视频搜索 / 后续精选视频推荐 |
| 固定计划不够灵活 | 生理期或临时忙碌后计划容易中断 | 后续引入请假与智能重排机制 |
| 周期记录与训练决策脱节 | 知道自己处于某个周期阶段，但不知道如何调整训练 | 将周期阶段转化为训练强度和内容建议 |
| 缺少解释与安全边界 | 训练建议不说明原因，用户不确定是否适合自己 | 输出推荐理由、安全提醒和替代方案 |

## 4. 竞品洞察摘要

HerFit 的竞品分析重点参考了 Strong、FitrWoman 和 Flo 三类产品。

| 竞品 | 核心特点 | 可借鉴点 | 对 HerFit 的启发 / 空白 |
|---|---|---|---|
| Strong | 强训练记录工具，适合力量训练和数据追踪 | 训练记录清晰、动作与组数结构化 | 不关注女性周期和身体状态，记录能力强但不解决“今天怎么练” |
| FitrWoman | 将周期阶段与运动建议结合 | 周期化训练具有明确用户价值 | 更偏专业运动员，日常女性健身场景和中文内容生态适配不足 |
| Flo | 强周期记录、症状追踪和健康内容 | 症状记录、周期可视化和内容推荐能力成熟 | 健康记录强，但不直接转化为具体训练决策 |

HerFit 的差异化方向是：

> 不只是记录训练，也不只是记录周期，而是把「周期阶段 × 身体状态 × 训练场景」转化为当天训练决策。

## 5. 产品定位与核心逻辑

### 5.1 产品定位

HerFit 的核心定位是：

> 女性周期友好的 AI 日常训练决策助手。

它不是单纯的训练记录工具，也不是单纯的经期记录工具，而是帮助用户在每天打开产品时快速回答：

> 今天适合练什么？应该练多强？如果状态不好，有没有更轻松的替代方案？

### 5.2 核心差异化逻辑

HerFit 的推荐逻辑基于三个核心维度：

> **周期阶段 × 身体状态 × 训练场景**

1. 周期阶段：月经期、卵泡期、排卵期、黄体期、未知 / 不规律周期
2. 身体状态：精力充沛、一般、很累、疲劳、腹胀、痛经、压力大、睡眠差等
3. 训练场景：训练目标、可用时间、训练地点、器械条件、跟练需求等

### 5.3 产品闭环

当前 MVP 已形成初步闭环：

```text
Onboarding 信息收集
        ↓
今日状态选择
        ↓
周期友好训练推荐
        ↓
搜索跟练视频 / 执行训练
        ↓
训练完成反馈
        ↓
训练记录沉淀
```

未来版本将进一步把训练记录、周期数据和用户反馈用于后续训练计划调整。

## 6. 当前 V0.1 MVP 范围

当前版本主要围绕“今日训练核心链路”展开。

### 6.1 已覆盖功能

| 模块 | 当前功能 |
|---|---|
| 新手引导 | 收集训练目标、训练水平、身体数据、周期信息和当前状态 |
| 今日训练 | 展示当前周期阶段，支持用户选择当天状态，生成今日训练建议 |
| 视频入口 | 根据训练主题跳转 B 站搜索结果，降低用户寻找跟练内容的成本 |
| 训练反馈 | 用户完成训练后记录感受和完成情况 |
| 周期视图 | 展示当前周期阶段、本周期关键日期和阶段训练建议 |
| 训练记录 | 展示完成次数、累计时长和最近训练记录 |
| 设置 | 支持查看 / 调整周期信息、训练偏好，并展示安全声明 |

### 6.2 当前不包含内容

当前 MVP 暂不包含：

- 真实大模型 API 调用
- 账号系统与云端同步
- 医疗级健康判断
- 长期历史数据驱动的个性化模型
- 自动抓取或嵌入 B 站视频
- 完整周 / 月训练计划生成
- 可穿戴设备数据接入

## 7. 核心用户流程

### 7.1 新用户首次使用流程

1. 用户进入 HerFit。
2. 完成 Onboarding，输入训练目标、训练水平、身体数据、周期信息和当前状态。
3. 系统根据基础信息建立初始推荐上下文。
4. 用户进入今日训练页面。

### 7.2 每日训练流程

1. 用户打开今日训练页面。
2. 系统展示当前周期阶段。
3. 用户选择今天状态：精力充沛 / 一般 / 很累。
4. 系统根据周期阶段和身体状态生成今日训练建议。
5. 用户查看训练内容、推荐理由和安全提醒。
6. 用户点击搜索跟练视频，跳转到 B 站相关搜索结果。
7. 用户完成训练后提交反馈。
8. 系统将训练结果沉淀到训练记录中。

### 7.3 训练记录流程

1. 用户进入训练记录页面。
2. 查看完成次数、累计时长和最近训练记录。
3. 训练记录作为后续个性化推荐和计划调整的数据基础。

## 8. 功能需求

### 8.1 Onboarding 新手引导

目标：收集生成个性化推荐所需的基础上下文。

核心字段：

- 训练目标
- 当前训练水平
- 身体基础信息
- 生理周期信息
- 当前身体状态
- 训练偏好

验收标准：

- 用户可以完成多步引导流程。
- 引导流程结束后进入主产品页面。
- 用户输入信息能够作为今日训练推荐的基础上下文。

### 8.2 今日训练页面

目标：帮助用户判断当天适合进行什么训练。

核心功能：

- 展示当前周期阶段
- 支持用户选择今日状态
- 展示今日训练主题、时长、动作和训练说明
- 提供推荐理由和安全提醒
- 提供搜索跟练视频入口
- 支持训练完成后的反馈记录

验收标准：

- 用户选择状态后，可以看到对应训练建议。
- 不同状态应影响训练强度或训练内容展示。
- 用户可以完成训练反馈。

### 8.3 周期视图

目标：帮助用户理解当前所处周期阶段及其对应训练建议。

核心功能：

- 展示当前周期阶段
- 展示本周期关键日期
- 展示不同周期阶段的训练建议

验收标准：

- 用户可以看到当前周期状态。
- 页面能解释不同阶段适合的训练方向。

### 8.4 训练记录

目标：沉淀用户训练结果，为后续个性化推荐提供基础。

核心功能：

- 训练完成次数
- 累计训练时长
- 最近训练记录
- 单次训练反馈

验收标准：

- 用户完成训练反馈后，记录页面能够展示最新训练结果。
- 记录信息能够支持后续计划调整的产品逻辑。

### 8.5 设置与安全声明

目标：让用户管理个人训练偏好，并明确产品边界。

核心功能：

- 周期信息展示 / 调整
- 训练偏好展示 / 调整
- 安全声明展示

验收标准：

- 用户可以查看主要偏好设置。
- 页面明确说明 HerFit 不替代专业医疗建议。

## 9. AI 推荐逻辑摘要

HerFit 的 AI 推荐逻辑采用：

> **规则型安全过滤 + AI 生成式训练推荐**

当前 MVP 阶段主要通过预设规则和 mock recommendation logic 验证推荐流程和交互体验。未来可接入真实 LLM API，用于生成更自然的推荐解释、训练计划调整和科普内容。

### 9.1 输入

- 周期阶段
- 身体状态
- 训练目标
- 可用时间
- 偏好强度
- 训练地点
- 可用器械

### 9.2 处理逻辑

1. 先进行安全过滤，判断是否需要降低强度或建议休息。
2. 再根据周期阶段调整训练方向。
3. 最后结合用户训练目标和场景生成训练建议。

### 9.3 输出

- 今日推荐训练类型
- 推荐强度
- 建议时长
- 动作组合
- 推荐理由
- 安全提醒
- 替代方案

详细 AI 逻辑见：`AI_LOGIC.md`。

## 10. 技术实现摘要

| 层级 | 当前方案 | 说明 |
|---|---|---|
| 前端 | React / TypeScript / Tailwind CSS / Vite | 移动端优先的 Web MVP |
| 数据 | localStorage / mock data | 用于验证流程和本地记录 |
| AI 推荐 | 预设规则 + mock recommendation logic | 当前阶段验证产品逻辑，未来可接入 LLM API |
| 部署 | Vercel | 用于在线 Demo 展示 |
| 开发方式 | ChatGPT / Codex / Cursor 辅助开发 | 使用 Prompt 文档与 Tech Guide 约束 AI 协作开发 |

### 10.1 视频推荐策略

当前版本采用 B 站搜索关键词跳转，不爬取、不嵌入视频，降低版权和维护风险。

后续版本计划建立精选视频库，为视频增加结构化标签，包括：

- 训练类型
- 训练强度
- 时长
- 器械需求
- 是否低冲击
- 适合周期阶段
- 不适合情况

再根据用户状态推荐 1-3 个具体跟练视频。

## 11. 后续迭代路线

### V0.2｜周期 × 训练记录日历

- 增加日历视图，将生理周期、症状、出血情况和训练记录整合在同一界面中。
- 支持用户实时添加每日身体状态、训练完成情况和备注。
- 为后续个性化推荐和计划调整建立数据基础。

### V0.3｜周 / 月训练计划视图

- 从单日训练推荐扩展为一周 / 一个月训练计划展示。
- 展示每天的训练类型、强度、时长、完成状态和周期适配说明。
- 帮助用户理解每日训练如何服务于更长期的训练目标。

### V0.4｜请假与智能重排

- 支持用户临时请假或跳过当天训练。
- 用户回归后，根据最新周期阶段、身体状态和训练目标重新调整后续计划。
- 提供轻量恢复、跳过继续、顺延计划等重排策略。

### V0.5｜精选视频推荐

- 建立带有结构化标签的 B 站精选训练视频库。
- 根据训练类型、强度、时长、器械和用户身体状态匹配具体视频。
- 推荐 1-3 个可直接跳转的跟练视频，降低用户筛选成本。

### V0.6｜科普知识增强

- 增加与推荐结果相关的轻量科普内容。
- 解释不同周期阶段、身体状态与训练强度之间的关系。
- 通过“为什么这样推荐”提升用户对 AI 建议的理解和信任。

## 12. 产品边界与风险

### 12.1 产品边界

HerFit 不是医疗产品，也不能替代医生、康复师或专业教练的建议。

HerFit 不提供医学诊断，不声称可以治疗任何经期症状或健康问题。

如果用户出现严重疼痛、头晕、异常出血、持续不适或其他异常症状，应停止训练并寻求专业医疗建议。

### 12.2 核心风险与应对

| 风险 | 影响 | 应对策略 |
|---|---|---|
| 训练建议不适合用户当天状态 | 高 | 引入规则型安全过滤，状态较差时降低强度或建议休息 |
| 用户误解为医疗建议 | 高 | 在设置页、推荐页和文档中明确免责声明 |
| 视频推荐质量不可控 | 中 | 当前使用搜索跳转，后续建立人工精选视频库和标签体系 |
| 本地数据丢失 | 中 | 当前 MVP 使用 localStorage，后续可引入云端同步 |
| 周期预测不准确 | 中 | 支持用户手动更新周期信息，不强行假设不规律周期 |
| AI 输出不可控 | 中 | 使用 Prompt 约束、输出结构约束和安全边界说明 |

## 13. 当前阶段成功指标

当前阶段不是商业化增长阶段，重点验证产品方向和使用闭环。

可观察指标包括：

- 用户是否能顺利完成 Onboarding
- 用户是否理解今日训练推荐的原因
- 用户是否愿意点击跟练视频入口
- 用户是否愿意完成训练反馈
- 用户是否能理解周期阶段与训练建议之间的关系
- 用户是否认为“周期 × 状态 × 场景”的推荐逻辑有价值

后续如果进入真实用户测试，可进一步观察：

- 7 日留存率
- 每周训练完成率
- 漏练后回归率
- 推荐满意度
- 用户对安全提醒和推荐解释的信任度
